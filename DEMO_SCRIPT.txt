[0:00 — HOOK]
Hello everyone. Reading and implementing academic research papers can be extremely daunting for students and beginners. They often get stuck on complex terminology, struggle to identify what prerequisites they need to study first, and have difficulty estimating the implementation complexity of models.

[0:20 — WHAT IT IS]
That is why we built Research Navigator AI. It belongs to the Agents for Good track and acts as an intelligent research mentor. Rather than just summarizing a paper, it helps students evaluate if they are ready to build it, maps out concrete learning paths, identifies datasets, and proposes multi-level project ideas.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram.
First, the user inputs their query, which can be an abstract, a topic, or an arXiv URL.
This goes straight to the Security Checkpoint node. If it detects prompt injections or out-of-domain chat queries, it triggers a warning and routes to the Security Alert terminal node to save API quota.
If it passes, it moves to the Coordinator Agent, which orchestrates six specialized sub-agents:
- The Paper Analyzer summarizes the core objectives and methodology.
- The Prerequisite Analyzer maps out required knowledge and learning times.
- The Implementation Analyzer estimates code complexity and timelines.
- The Dataset Analyzer checks dataset sizes and looks up public alternatives.
- The Project Idea Generator proposes beginner, intermediate, and advanced practical project ideas.
- And the Research Mentor performs a feasibility assessment against the user's skill level.
Once collected, these findings are combined into a consolidated report. We then pause at the Human Review node, where the user can approve the recommendation or provide feedback to refine it, before outputting the finalized report.

[3:00 — SECURITY & MCP]
Under the hood, our security checkpoint automatically scrubs personal emails and phone numbers, checks for injection attempts, and writes structured JSON logs of every event to ensure complete auditing. Our custom MCP server exposes tools like check_prerequisites, check_dataset_spec, and estimate_complexity to ground the agents' knowledge in verified local datasets.
