[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram.
First, the user inputs their query, which can be an abstract, a topic, or an arXiv URL.
This goes straight to the Security Checkpoint node. If it detects prompt injections, shell/script execution attempts, or out-of-domain queries, it triggers a warning and routes to the Security Alert terminal node to save API quota.
If it passes, it moves to the Coordinator Agent, which orchestrates five consolidated sub-agents sequentially to keep requests safely under the Gemini API rate limits:
- The Paper Analyzer summarizes the core objectives and methodology.
- The Prerequisite & Learning Path Analyzer maps out required knowledge, learning times, and weekly roadmap.
- The Implementation, Dataset, & Research Impact Analyzer estimates code complexity, hardware, dataset specifications, and domain relevance.
- The Project Idea Generator proposes beginner, intermediate, and advanced practical project ideas.
- And the Readiness, Feasibility, & Mentorship Agent performs overall readiness and suitability assessments.
Once collected, these findings are combined into a consolidated report. We then pause at the Human Review node, where the user can approve the recommendation or provide feedback to refine it, before outputting the finalized report.

[3:00 — SECURITY & MCP]
Under the hood, our security checkpoint automatically scrubs personal emails and phone numbers, checks for injection attempts, blocks remote shell code execution vectors, and writes structured JSON logs of every event to ensure complete auditing. Our custom MCP server exposes seven tools including get_learning_path, get_project_templates, get_research_domain, and get_readiness_rules to ground the agents' knowledge in verified local datasets.
