Hello everyone. Reading and implementing academic research papers can be extremely daunting for students and beginners. They often get stuck on complex terminology, struggle to identify what prerequisites they need to study first, and have difficulty estimating the implementation complexity of models.

[0:20 — SOLUTION OVERVIEW]
That is why we built Research Navigator AI. It acts as an intelligent research mentor. Rather than just summarizing a paper, it helps students evaluate if they are ready to build it, maps out concrete learning paths, identifies datasets, and proposes multi-level project ideas.

[0:40 — SOLUTION ARCHITECTURE]
(Show assets/architecture_diagram.png)
Let's walk through our optimized single-agent coordinator workflow. 
First, the user inputs their query (an abstract, topic, or arXiv URL), which passes through our Security Checkpoint node to filter out prompt injections, shell/script execution attempts, and out-of-domain queries.
If it passes, it moves to the Coordinator Agent. The Coordinator interfaces directly with 7 local MCP database tools (such as check_prerequisites, estimate_complexity, and get_learning_path). Since these tools execute locally, the Coordinator fetches all grounded context instantly and compiles the complete 10-section mentoring report in a single LLM request, avoiding API rate limits.
The workflow then pauses at the Human Review node, where the user can approve the recommendation or provide feedback to refine it, before outputting the finalized report.
